# Prompt Engineering

## 什么是Prompt Engineering？
通过精心设计输入文本，引导LLM产生期望的输出。不需要修改模型参数。

## Prompt的五要素

### 1. System Prompt（角色设定）
- 定义模型的角色、能力边界
- 示例："你是一个专业的技术面试辅导助手，专精于LLM和大模型领域"

### 2. Context（上下文/参考资料）
- RAG检索到的相关文档放这里
- 提供模型本来不知道的信息

### 3. Instruction（指令）
- 明确告诉模型要做什么
- "请用中文回答，控制在200字以内"
- "先分析问题，再给出答案"

### 4. Few-shot Examples（少样本示例）
- 给2-3个输入→输出的范例
- 性价比最高的prompt技巧
- 格式一致性比内容更重要

### 5. Output Format（输出格式约束）
- JSON格式：适合程序解析
- Markdown表格：适合展示结构化信息
- "只输出是或否"：二分类场景

## 进阶技巧

### CoT (Chain-of-Thought)
- "让我们一步步思考"
- 引导模型展示推理过程
- 数学和逻辑题效果显著

### ReAct (Reasoning + Acting)
- 思考 → 行动 → 观察 → 思考...
- Agent的核心循环
- "我需要先查X，然后根据结果决定下一步"

## 常见陷阱
- 指令冲突：System说"简洁"，User说"详细" → 模型困惑
- 过度约束：太多规则反而导致模型忽略关键指令
- 幻觉诱导：问"请列出99个原因" → 模型会编造
