# RLHF 与 DPO —— 对齐训练

## 为什么需要对齐？
预训练模型学会了"续写"，但不会"按人类偏好回答"。
对齐 = 让模型输出符合人类价值观（有帮助、诚实、无害）。

## RLHF (Reinforcement Learning from Human Feedback)

### 三步流程
1. **SFT (监督微调)**：用高质量(指令,回答)对训练模型学会问答格式
2. **训练奖励模型(RM)**：让人标注"哪个回答更好"→ 训练一个模型来打分
3. **PPO强化学习**：用奖励模型的打分作为信号，优化策略模型

### RLHF的问题
- 需要训练和维护一个独立的奖励模型
- PPO训练不稳定，需要大量调参
- 流程复杂，工程成本高

## DPO (Direct Preference Optimization)

### 核心思想
跳过奖励模型，直接用(好回答, 坏回答)对来优化模型。

### 数学公式
Loss = -log σ(β × (log P_θ(好|prompt) / P_ref(好|prompt) - log P_θ(坏|prompt) / P_ref(坏|prompt)))

- β: 温度参数，控制偏离参考模型的程度
- P_θ: 当前模型的概率
- P_ref: 参考模型(SFT后的模型)的概率
- 直观理解：让好回答的概率上升，坏回答的概率下降

### DPO vs RLHF
| | RLHF | DPO |
|---|---|---|
| 组件 | 4个模型(SFT+RM+Policy+Ref) | 2个模型(Policy+Ref) |
| 训练稳定性 | 不稳 | 稳 |
| 数据需求 | 偏好标注 | 偏好标注（同RLHF） |
| 性能 | SOTA | 接近RLHF |

## 实际应用
- 情感陪伴产品中，需要让模型"暖心但不越界"
- 客服场景中，需要"专业但不冷漠"
- DPO的核心在于构造高质量的好坏回答对
